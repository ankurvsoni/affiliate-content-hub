## Learnings captured (2026-03-07 late session)
- For multi-project domains on Vercel, deployments can be correct but aliases may still point to older builds; always verify and explicitly re-alias after critical UI/content fixes.
- Keep one canonical GitHub repo per project from day one to avoid Vercel mirror-repo confusion during reconnects.
- Add buyer-facing wording QA before publish (remove internal terms like "prompt set", "human-style", and publisher-centric FAQ items).
- For affiliate UX, inline product links must be visually obvious; pill/button styling significantly improves discoverability over plain links.
- Amazon compliance baseline to enforce on every affiliate project:
  - "As an Amazon Associate I earn from qualifying purchases."
  - clear disclosure page + trademark line
  - avoid static exact prices in article body unless reliably refreshed
  - product-specific Amazon links (ASIN + tag), avoid generic Amazon homepage CTAs.
- Cross-project navigation should be explicit in top nav (e.g., "All Projects") and verified on live pages, not assumed from code.
- After wording/compliance updates, run live HTML checks for banned terms and legacy phrases across all project subdomains.
