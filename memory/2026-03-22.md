## Infrastructure Map (CRITICAL — read before any deploy)

### Two separate repos, two separate Vercel projects, two separate sites

| Site | GitHub Repo | Vercel Project | Notes |
|---|---|---|---|
| `www.orkolabs.com` | `ankurvsoni/ai-writing-tools-hub` | `ai-writing-tools-hub` | Project hub only. Simple codebase, no content/pages, no affiliate system |
| `reviews.orkolabs.com` | `ankurvsoni/affiliate-content-hub` | `affiliate-content-hub` | Affiliate content site. Has content/pages/, product JSON, affiliate system |

### Workspace mapping
- `/Users/orko/.openclaw/workspace-income-factory` = **affiliate-content-hub** (reviews.orkolabs.com)
- `ai-writing-tools-hub` changes must be done via `/tmp` clone of that repo separately

### Deploy checklist for reviews.orkolabs.com
1. Work in `/Users/orko/.openclaw/workspace-income-factory`
2. `git push affiliatehub HEAD:main`
3. `npx vercel link --yes --project affiliate-content-hub --scope ankurvsonis-projects`
4. `npx vercel --prod --yes --build-env NEXT_PUBLIC_SITE=reviews` ← MUST pass build-env
5. Get the deployment URL from output (e.g. `affiliate-content-XXXXX-ankurvsonis-projects.vercel.app`)
6. `npx vercel alias set <deployment-url> reviews.orkolabs.com --scope ankurvsonis-projects` ← MUST do this every time
7. Verify both `reviews.orkolabs.com` and the deployment URL load correctly

### Deploy checklist for www.orkolabs.com
1. Clone `https://github.com/ankurvsoni/ai-writing-tools-hub.git` to `/tmp`
2. Make changes there
3. `git push origin main`
4. `npx vercel link --yes --project ai-writing-tools-hub --scope ankurvsonis-projects`
5. `npx vercel --prod --yes`
6. Vercel auto-aliases to `www.orkolabs.com`

### Critical lessons learned
- `reviews.orkolabs.com` alias is NOT automatic — it must be manually set after every affiliate-content-hub deploy
- `NEXT_PUBLIC_SITE` is a build-time env var — must pass via `--build-env` flag, not just project env settings
- Never cross-push between the two repos — they are independent codebases
- After any deploy, verify BOTH the raw Vercel deployment URL AND the custom domain

## Affiliate Content Hub Architecture (reviews.orkolabs.com)

### Content system
- `content/pages/[slug].md` — page prose, metadata, editorial sections
- `content/data/products/[slug].json` — structured product data (separate from prose)
- Products injected via `{{products}}` token in markdown body
- `lib/content.js` merges both at build time

### Adding/updating a product
Edit `content/data/products/[slug].json` only — no markdown surgery needed:
```json
{
  "slug": "page-slug",
  "affiliate_tag": "triedandtr0d5-20",
  "products": [
    {
      "name": "Product Name",
      "asin": "B0XXXXXXXXX",
      "confidence": "medium",
      "tagline": "one-line descriptor",
      "what_it_is": "...",
      "best_for": "...",
      "stands_out": "...",
      "before_you_buy": "...",
      "skip_if": "..."
    }
  ]
}
```

### Adding a new page
1. Create `content/pages/[slug].md` with full frontmatter + prose
2. Create `content/data/products/[slug].json` with product picks
3. Add slug to `allowed` set in `app/all-guides/page.js`
4. Add to featured guides in `app/page.js` (ReviewsHomePage) if trending

### All-guides allowed list
Located in `app/all-guides/page.js` — slugs must be explicitly added to `allowed` Set to appear.

### Amazon affiliate tag
`triedandtr0d5-20` — all product links use this tag
